// what this is

This is a folder structure. A set of markdown conventions. A discipline you impose on your file system and then operate deliberately.

It is not a SaaS product. It doesn't sync, notify, or integrate with anything. There is no dashboard. There is no AI assistant embedded in it. There is no vendor to negotiate with.

A product leader using this system maintains a small set of files that answer the questions everyone actually asks: What are we focused on? Who owns what? What did we decide and why?

The system is file-based and GitHub-native. You clone it, rename the folders to match your org, and start writing. That's the whole setup.

This is a prototype. The system is evolving. If you find it useful, the right response is to fork it and make it yours.

// why this exists

FOUR PROBLEMS
NAMED DIRECTLY.

01

Context fragmentation.
The strategy is in a deck presented once. The decisions are in Slack. The team's focus is in someone's memory. Nothing is in one place, and the cost is paid every time someone needs to make a call.

02

No operating system.
You have tools. You don't have a system. Tools without a system produce noise, not signal. Jira doesn't tell you what you're focused on. Notion doesn't tell you what you decided.

03

Decisions disconnected from context.
A decision log without the context that produced it is a list of outcomes. It doesn't help anyone reason about the next decision. You need the why, not just the what.

04

No visibility without a meeting.
Leaders shouldn't need to schedule 30 minutes to understand where their org is. That information should be readable in under two minutes, any time, without asking anyone.

// how it works

THE CADENCE
SYSTEM.

daily

Skim now.md. It tells you today's focus and the active blocker. Update it when something changes. It's one file, always in the same place.

weekly

Update team sprint files. Pull blockers forward. Generate the biweekly update from the decisions log and team files. It's a 15-minute operation if the files are current.

quarterly

Write the quarterly memo. Four sections: what we shipped, what we didn't, what changed in our thinking, what's next. It takes 90 minutes. It replaces a QBR deck.

on demand

decisions/ is queryable. Grep it. Sort by date. Filter by area. When someone asks "why did we do that?" — you open a file. You don't interrupt anyone. You don't schedule a call.
